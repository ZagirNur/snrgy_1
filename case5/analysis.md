# Аналитический обзор (по Кейсу 4)

Разбираю своё веб-приложение "Турфирма" (папка case4) по основным критериям.

## Функциональность
Приложение умеет делать все базовые вещи: показать список туров, добавить новый тур, посмотреть подробности, удалить.

## Производительность
Данных мало, поэтому работает быстро. Если туров станет много, начнёт тормозить запрос списка без индексов. Можно добавить индексы.

## Удобство использования
Интерфейс совсем простой: голый HTML без стилей. Можно подключить Bootstrap.

## Безопасность
SQL-инъекций нет, потому что везде используются параметризованные запросы через sqlite3. Но нет авторизации - любой может удалить тур. Нет CSRF-защиты.

## Масштабируемость
SQLite - файловая база, для одного пользователя подходит, но при большом числе одновременных запросов начнутся проблемы с блокировками. Решение - перейти на PostgreSQL.

## Сопровождаемость
Код небольшой и весь в одном файле app.py, поэтому пока читается нормально. Когда функций станет больше, надо будет разбить на модули (views, models, templates) и использовать Blueprint.

## Переносимость
Всё написано на Python + Flask + SQLite, работает на любой ОС где есть Python 3. Переносить легко - достаточно скопировать папку и выполнить pip install -r requirements.txt.

## Качество кода
Код простой, имена переменных короткие, но понятные. Нет обработки ошибок (если передать не число в поле цены - упадёт). Решение - добавить try/except и валидацию форм.

## Тестирование
Тестов нет. Проверял руками через браузер. Решение - написать несколько unit-тестов на pytest + Flask test client для основных роутов.

## Итог
Приложение решает учебную задачу, но для продакшена нужно добавить авторизацию, CSRF-защиту, валидацию, тесты, стили и переход на PostgreSQL.
